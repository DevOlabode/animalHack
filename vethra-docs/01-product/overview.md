## Feature spec template (used in each file)

Every product doc includes:

1. Objective  
2. Phase  
3. User stories  
4. Acceptance criteria  
5. Business rules  
6. Validation rules  
7. Edge cases  
8. Permissions  
9. Data model references  
10. API references  
11. UI requirements  
12. Error states  
13. Future considerations
