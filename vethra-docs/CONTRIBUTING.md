# Contributing

## Source of truth

Implement against `vethra-docs/`. If code and docs disagree, update docs in the same PR/commit when changing behavior intentionally.

## Workflow

1. Read `12-hackathon/mvp.md` before adding features
2. Pick one feature doc under `01-product/`
3. Implement API → model → UI in that order when possible
4. Run acceptance checks in `10-testing/acceptance-tests.md`
5. Do not merge Future-phase features into MVP navigation

## PR hygiene (post-hackathon)

- Small PRs
- Note phase label (MVP / Phase 2 / Future)
- Include screenshots for UI changes
